# usb-dfu 模式（usb-dfu / USB DFU）

此模式通过 USB DFU 引导加载器烧录固件（STM32 / GD32，依赖 WebUSB/DFU 相关实现）。

## 准备条件

- 目标板必须具备可工作的 USB DFU 引导加载器。
- 让目标板进入 DFU 模式：
  - 依照你的板卡说明设置 `BOOT0=1`（或使用板子上的 DFU 进入方式）
  - 复位进入 DFU
- 浏览器需支持 WebUSB（建议 Chromium 系列，且需要安全上下文）。

## 连接方式

- 连接目标板的**原生 USB 接口**到电脑（以常见 STM32 USB 引脚为例，可能对应 `PA11/PA12`，以实际电路为准）。
- **注意供电与共地**：USB 自带电源一般无需额外供电，但确保硬件连接正常即可。

## 网页操作步骤

1. 在主页选择模式为 `usb-dfu`（对应 `usb-dfu` 指南）。
2. 选择固件文件（支持 `*.hex` / `*.bin`）。
3. 点击「下载固件」。
4. 浏览器会弹出设备选择/权限请求，选择目标板的 DFU 设备并确认。
5. 等待进度条完成；成功后通常可以切回运行模式（如果需要由目标板重新进入运行状态）。

## 常见问题

- **浏览器里找不到 DFU 设备**
  - 确认目标板确实已进入 DFU 模式（BOOT0/复位流程正确）。
  - Windows 上通常需要为 DFU 接口安装合适的驱动（常用 `Zadig` 安装 WinUSB/libusb 驱动）。

- **选择了设备但下载失败**
  - 检查你使用的固件类型/格式是否与板子的 DFU bootloader 兼容。
  - 可能需要先切换运行模式与 DFU 模式（不同 DFU 实现流程略有差异）。

## 浏览器要点

- 需要安全上下文：HTTPS 或 localhost。
- WebUSB 在 Chromium 系列浏览器更可靠。

## 驱动要点（Windows 常见）

- 使用 `Zadig` 为 DFU 接口安装 WinUSB/libusb 兼容驱动。
- 如不确定如何选对设备，先确认 DFU 模式下设备的厂商/产品信息再安装。
