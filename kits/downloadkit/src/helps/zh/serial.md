# serial 模式（serial / UART）

此模式使用芯片 ROM 串口系统引导加载器（UART bootloader）烧录固件，适用于 STM32 / GD32 系列。

## 准备条件

- 浏览器支持 `Web Serial API`（建议 Chrome / Edge / Arc，且必须是安全上下文）。
- 主控端到设备端之间的串口驱动已安装（USB 转串口：CH340 / CP2102 / FT232 等）。
- 让目标板进入 ISP/UART 引导加载器模式：
  - 按板子手册设置 `BOOT0=1`
  - 复位目标板

## 连接方式

通常连接如下（以常见 STM32F1 为例）：

- `TX`（目标板） -> `RX`（USB 转串口）
- `RX`（目标板） -> `TX`（USB 转串口）
- **GND 必须共地**

> 如果你的电路/引脚定义不同，请按目标板原理图调整。

## 网页操作步骤

1. 在主页选择模式为 `serial`。
2. 选择固件文件（支持 `*.hex` / `*.bin`）。
3. 在下方选择波特率（默认 `115200`，按你目标板的 bootloader 配置选）。
4. 点击「下载固件」。
5. 浏览器会弹出串口设备授权选择框，选择对应的串口设备并确认。
6. 等待进度条完成；成功后页面会提示烧录完成。

## 常见问题

- **点了下载但找不到端口**
  - 检查 USB 转串口驱动是否安装正确
  - 确认目标板确实已进入引导模式（`BOOT0=1` + 复位）
  - 确认波特率是否匹配

- **握手失败 / 超时**
  - 复位目标板后重试
  - 确认 RX/TX 是否交叉接对、GND 是否共地
  - 若板上 bootloader 版本使用不同协议/参数，需要匹配固件与配置

## 浏览器要点

- 需要安全上下文：HTTPS 或 localhost。
- 仅支持 Chromium 系列浏览器的 Web Serial。

## 驱动要点（Windows 常见）

- CH340 / CP2102 / FT232 等设备在 Windows 上需要对应驱动。
- 若设备识别异常（出现未知设备），先修复驱动后再尝试 Web Serial 授权。
